Figma to Code with Claude Code & Material UI

A practical workflow for converting Figma designs to production-ready React code

Prerequisites Install Tools Create Project Configure Claude Connect Figma Workflow Common Issues Best Practices Quick Reference

Prerequisites

Operating System

macOS 10.15 or later

Subscription

Claude Pro or Max ($20-100/month)

Design Tool

Figma Desktop App (not web version!)

Development

GitHub account & VS Code

Part 1: Install Development Tools

1. Install Homebrew (Mac Package Manager)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

After installation, run the commands it shows you (usually):

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

Verify:

brew --version

2. Install Node.js

brew install node

Verify:

node --version
npm --version

3. Install GitHub CLI

brew install gh

Authenticate:

gh auth login
# Choose: GitHub.com → HTTPS → Login with browser

4. Install Claude Code CLI

curl -fsSL https://claude.ai/install.sh | bash

Reload your shell:

source ~/.zshrc

If .zshrc doesn't exist:

touch ~/.zshrc
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

Verify:

claude --version

Part 2: Create Your React Project

1. Create Project with Vite + TypeScript

# Navigate to your projects folder
cd ~/Documents  # or wherever you keep projects

# Create new React project
npm create vite@latest my-mui-project -- --template react-ts

# Navigate into it
cd my-mui-project

# Install dependencies
npm install

2. Install Material UI v6

npm install @mui/material @emotion/react @emotion/styled

3. Initialize Git and Push to GitHub

# Initialize git
git init
git add .
git commit -m "Initial commit: React + TypeScript + Vite"

# Create repo on GitHub (opens browser)
gh repo create my-mui-project --public --source=. --remote=origin --push
Alternative manual method:
  1. Go to https://github.com/new
  2. Create repository (don't initialize with README)
  3. Run the commands GitHub shows you

Set git identity (recommended):

git config --global user.name "Your Name"
git config --global user.email "your.email@gmail.com"

Part 3: Configure Claude Code

1. Create Project Context File

mkdir .claude
code .claude/CLAUDE.md  # or: nano .claude/CLAUDE.md

2. Add This Content to CLAUDE.md

# Project Context

## Tech Stack
- React 18 with TypeScript
- Vite for build tooling
- Material UI v6 (@mui/material)

## Design System Rules

### Component Library
- ALWAYS use Material UI v6 components from '@mui/material'
- Never create custom buttons, inputs, cards when MUI has them
- Check MUI docs first before creating any UI component

### Design Token Mapping (Figma → MUI)
- Colors: Map to MUI theme palette (primary, secondary, error, etc.)
- Spacing: Use theme.spacing() - 1 unit = 8px
- Typography: Map to MUI variants (h1-h6, body1, body2, button)
- Borders: Use theme.shape.borderRadius
- Shadows: Use theme.shadows[0-24]

### Component Patterns
- Buttons: `<Button variant="contained|outlined|text">`
- Layout: `<Container>`, `<Grid>`, `<Stack>`, `<Box>`
- Forms: `<TextField>`, `<Select>`, `<Checkbox>`
- Cards: `<Card>` with `<CardContent>`, `<CardActions>`

## Code Style
- Use TypeScript for all files
- Define Props interfaces above components
- Use `sx` prop for styling with theme values
- Never hardcode colors, spacing, or font sizes
- Always make components responsive

## File Organization
```
src/
  components/     # Reusable UI components
  pages/         # Page-level components
  theme.ts       # MUI theme configuration
```

## Figma Integration
When converting Figma designs:
1. Use Figma MCP to fetch actual design data
2. Extract design tokens first (colors, spacing, typography)
3. Map Figma components to MUI equivalents
4. Preserve visual hierarchy and spacing
5. Use sx prop to match exact measurements
6. Create theme overrides for custom colors

Save and exit (nano: Ctrl+O, Enter, Ctrl+X)

3. Install VS Code Extension

  1. Open VS Code Extensions (Cmd+Shift+X)
  2. Search for "Claude Code"
  3. Install the official Anthropic extension
  4. Sign in with your Claude Pro/Max account when prompted

Part 4: Connect Figma MCP Server

1. Enable Figma MCP in Figma Desktop

  1. Open Figma Desktop App (must be desktop, not web!)
  2. Figma menu (top-left) → Preferences
  3. Find "Dev Mode MCP Server"
  4. Enable it
  5. Note the URL: http://127.0.0.1:3845/mcp
Important: Keep Figma Desktop running!

2. Add Figma MCP to Claude Code

cd ~/Documents/my-mui-project
claude mcp add figma-desktop --transport http -- http://127.0.0.1:3845/mcp

3. Add MUI MCP Server

claude mcp add mui-mcp -- npx -y @mui/mcp@latest

4. Verify MCP Connections

Restart VS Code, then in Claude Code panel, type:

/mcp

Should show:

  • ✅ figma-desktop: connected
  • ✅ mui-mcp: connected

Part 5: The Workflow

1. Start Your Dev Server

Open a new terminal tab (Cmd+T in Terminal, or click + in VS Code terminal):

cd ~/Documents/my-mui-project
npm run dev
Leave this running! Open new terminal tabs for other commands.

2. Create Design in Figma

  1. Open Figma Desktop
  2. Create your layout/component
  3. Switch to Dev Mode (toggle top-right)
  4. Select the frame you want to convert
  5. Copy the Figma file URL

3. Convert Figma to Code

Open Claude Code in VS Code:

  • Click the ⚡ spark icon (top-right when file is open)
  • Or: Cmd+Shift+P → "Claude Code: Open"

Example prompt:

I have a Figma design at [paste URL].

Please:
1. Fetch the design from Figma using the MCP
2. Extract all design tokens (colors, spacing, typography)
3. Convert it to React using Material UI v6 components
4. Match the visual design exactly
5. Create proper TypeScript types
6. Save as src/components/[ComponentName].tsx

Follow all rules in CLAUDE.md.

4. Review and Accept Changes

  • Claude shows diffs for each file change
  • Click Accept for changes you want
  • Click Reject for changes you don't want
  • Test in browser at http://localhost:5173

5. Commit and Push to GitHub

In a new terminal tab (not the one running dev server):

# Stage changes
git add .

# Commit with message
git commit -m "Add [component name] from Figma design"

# Push to GitHub
git push

First push might need:

git push -u origin main

Common Pitfalls & Solutions

"command not found: claude"

Solution: Add to PATH 
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

"command not found: npm"

Solution: Install Node.js first 
brew install node

"command not found: brew"

Solution: Install Homebrew (see Part 1, Step 1)

Can't paste GitHub token in terminal

Solution: Just paste (Cmd+V) - it won't show anything on screen (security feature), then press Enter

Figma MCP won't connect

Solutions:
  • Make sure Figma Desktop is running (not just web)
  • Verify Dev Mode MCP is enabled in Figma Preferences
  • Restart Figma Desktop
  • Make sure URL is exactly: http://127.0.0.1:3845/mcp

"site can't be reached" at localhost

Solution: Start the dev server 
npm run dev

Can't commit - terminal stuck

Solution: You're in the dev server terminal. Open a new terminal tab (Cmd+T) for git commands

Changes not showing in browser

Solutions:
  • Check console for errors (F12)
  • Make sure dev server is running
  • Hard refresh: Cmd+Shift+R
  • Clear browser cache

"fatal: The current branch main has no upstream branch"

Solution: First push needs: 
git push -u origin main

Terminal frozen in nano

Solution:
  • Save: Control+O, Enter
  • Exit: Control+X
  • (Use Control, not Command!)

Tips & Best Practices

Multiple Terminal Tabs

  • Tab 1: Dev server (npm run dev) - leave running
  • Tab 2: Git commands, Claude Code CLI, etc.

VS Code Built-in Git

Instead of terminal, use VS Code:

  1. Click Source Control icon (left sidebar)
  2. Stage changes with + icon
  3. Type commit message
  4. Click ✓ to commit
  5. Click ... → Push

Reference Files in Claude

  • Type @filename to give Claude context
  • Select code in editor - Claude sees it automatically

Always Review Diffs

  • Don't blindly accept all changes
  • Check that MUI components are used correctly
  • Verify TypeScript types are proper

Iterative Refinement

If Claude's output isn't perfect:

The spacing is too tight. Use theme.spacing(3) instead of 2.
The button should be variant="contained" not "outlined".

Keep Figma Organized

  • Name your frames clearly
  • Use Figma variables for colors
  • Document your design tokens
  • Keep components organized
